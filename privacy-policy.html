Privacy Policy

Effective date: March 10, 2026

MeloTab is a Chrome new tab extension. This policy explains what data the extension uses and how it is handled.

Summary

  • MeloTab does not sell user data.
  • MeloTab does not run ads.
  • MeloTab does not use analytics or tracking scripts.
  • MeloTab does not send personal data to a developer-controlled backend.
  • Most settings are stored locally in Chrome storage.

Data the Extension Uses

1. Settings and customization data

MeloTab stores interface settings such as theme, background choice, shortcuts, enabled widgets, selected search engine, AI launcher choices, todo items, and similar preferences in Chrome storage so the new tab page can keep the user's setup.

2. Weather data and optional location

If the user enables weather by current location, the extension may request the browser's geolocation permission. Location is used only to fetch weather forecast data from Open-Meteo. If the user does not allow location, MeloTab can use a city-based fallback instead.

3. Gmail unread count

If the user keeps a Gmail shortcut, MeloTab can request the Gmail Atom feed from mail.google.com to show an unread count badge. The extension reads the unread count only. Email message bodies are not stored by the extension and are not transmitted to the developer.

4. Calendar feeds

If the user adds a Google Calendar secret iCal URL, MeloTab fetches that feed directly from Google Calendar and parses upcoming events locally inside the extension. Calendar feed URLs are stored in Chrome storage so the widget can continue to work between sessions.

5. Search suggestions

When live search suggestions are enabled, the current query typed into the search box may be sent to the selected suggestion provider such as Google, Bing, DuckDuckGo, or Wikipedia to return suggestion results.

6. MeloDrop peer-to-peer file sharing

The optional MeloDrop feature uses public WebRTC trackers and STUN servers to help two browsers discover each other and establish a direct connection. Selected files are transferred directly between participating browsers after connection is established. The extension stores only local convenience data such as device name and last used room name.

What MeloTab Does Not Do

  • Does not collect browsing history
  • Does not inject scripts into websites
  • Does not read page content from arbitrary tabs
  • Does not upload user files to a developer server
  • Does not use third-party analytics or advertising SDKs

Data Sharing

MeloTab does not share user data with the developer for resale, profiling, advertising, or analytics. Data is sent only to the third-party services needed for the chosen feature to work:

  • Open-Meteo for weather and city lookup
  • Google, Bing, DuckDuckGo, or Wikipedia for search suggestions
  • Google Calendar for calendar feeds
  • Gmail for unread count
  • Public WebRTC trackers and STUN servers for MeloDrop peer discovery

Data Storage

Data is stored in the browser using Chrome extension storage and, where applicable, browser-managed local mechanisms. The user can clear this data by removing the extension or resetting their browser extension storage.

Security

MeloTab is designed to keep logic on-device where possible. The extension package contains its application code locally and does not download executable code from remote servers.
